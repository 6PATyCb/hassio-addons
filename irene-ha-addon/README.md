# irene-ha-addon
Аддон обертка для запуска [Ирины](https://github.com/janvarev/Irene-Voice-Assistant) в HA

## Как это работает?

Аддон выкачивает последнюю версию [Ирины](https://github.com/janvarev/Irene-Voice-Assistant) из Гитхаб репозитория и разворачивает ее в docker контейнере внутри экосистемы Home Assistant.
При развертывании происходит подключение `runva_webapi_docker.json` настроек Ирины как  `runva_webapi.json` и каталога `options_docker` как  `options`.

В момент первого запуска все исходники Ирины переносятся в каталог `/config/irene` и доступны к редактированию через стандартные механизмы HA такие как `Samba share` или `File editor`. После редактирования файлов нужно перезапустить аддон.

## Какие есть известные проблемы и решения

* Первый запуск может быть очень долгим т.к. из интернета скачивается образ докера и исходник Ирины. Следите за файлом лога.

* Если вам нужно сделать чистую установку, то вы должны самостоятельно сделать резервные копии файлов из каталога `/config/irene` (можно его переименовать в `/config/irene_old`), а удалить уже после того как он станет не нужен.

* Из-за того, что все исходники Ирины переносятся в каталог `/config/irene` при первом запуске, без чистой установки, аддон не сможет скачать последнюю версию Ирины, если вы уже давно пользуетесь аддоном, т.к. не будет знать какие файлы ему разрешено перезаписывать при обновлении.


## Как отлаживать самостоятельно
Для отладки в локальном докере выкачайте проект и соберите Dockerfile
```
docker build --build-arg BUILD_FROM=ghcr.io/hassio-addons/debian-base:stable -t irene-ha-addon .
```

Для запуска под виндой контейнера в локальном докере выполните команду (все исходники Ирины будут в вашем отдельном volume с именем `irene-config`):
```
docker run --rm -it ^
  --name irene-ha-addon ^
  -p 5003:5003 ^
  -v irene-config:/config ^
  -e HA_URL="http://ВАШ_IP_HA:8123/api" ^
  -e HA_TOKEN="ВАШ_ПРИВАТНЫЙ_ТОКЕН" ^
  -e LOG_LEVEL="debug" ^
  irene-ha-addon
```
